# 串口通信协议设计

本项目当前支持 ASCII 数值数据格式，例如 `0.5` 或 `0.1,0.2,0.3`。后续可以扩展为面向 STM32 下位机的数据帧协议，用于实时采集 ADC、温湿度、电压、电流等多通道数据。

## 数据帧格式

```text
$DATA,<seq>,<timestamp_ms>,<ch1>,<ch2>,<ch3>*<crc>\r\n
```

示例：

```text
$DATA,15,123456,25.60,60.20,3.31*7A
```

## 字段说明

| 字段             | 说明                         |
| ---------------- | ---------------------------- |
| `$DATA`        | 数据帧帧头                   |
| `seq`          | 帧序号，用于丢包检测         |
| `timestamp_ms` | 下位机启动后的毫秒时间戳     |
| `ch1/ch2/ch3`  | 多通道采样数据               |
| `crc`          | 校验字段，可使用 XOR 或 CRC8 |

## 解析流程

1. 串口接收到原始字节流。
2. 按 `\r\n` 分割完整帧。
3. 检查帧头是否为 `$DATA`。
4. 提取 `*` 前的数据字段和 `*` 后的校验字段。
5. 计算 CRC 并与帧内 CRC 比较。
6. 校验通过后转换为多通道采样数据。
7. 将采样数据送入波形缓存和绘图模块。

## 异常处理

| 异常情况     | 处理方式                        |
| ------------ | ------------------------------- |
| 帧头错误     | 丢弃该帧，解析失败计数 +1       |
| CRC 错误     | 不进入波形缓存，CRC 错误计数 +1 |
| 字段数量不足 | 丢弃该帧，记录错误日志          |
| 数值转换失败 | 丢弃该帧，避免通道错位          |
| 序号不连续   | 丢包计数 +1                     |
| 半包         | 暂存在缓冲区，等待下一批数据    |
| 粘包         | 按换行符拆分后逐帧解析          |

## 上位机命令帧

Qt 上位机后续可以下发控制命令，例如控制 LED、PWM 或采样周期。

格式：

```text
$CMD,<target>,<value>*<crc>\r\n
```

示例：

```text
$CMD,LED,1*2B
$CMD,PWM,500*4F
$CMD,PERIOD,20*61
```
