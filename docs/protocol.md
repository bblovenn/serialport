# 串口通信协议设计

项目当前兼容两类输入：

1. 普通 ASCII 数值格式，例如 `0.1,0.2,0.3`
2. 面向下位机的协议帧格式，例如 `$DATA,1,1000,0.20,0.30,0.40*17`

当前代码已经实现了 `$DATA` 协议帧解析、XOR 校验、错误提示和帧日志显示。

## 数据帧格式

```text
$DATA,<seq>,<timestamp_ms>,<ch1>,<ch2>,<ch3>*<crc>\r\n
```

示例：

```text
$DATA,15,123456,25.60,60.20,3.31*7A
```

## 字段说明

| 字段             | 说明                        |
| ---------------- | --------------------------- |
| `$DATA`        | 数据帧帧头                  |
| `seq`          | 帧序号，用于检测丢包        |
| `timestamp_ms` | 下位机启动后的毫秒时间戳    |
| `ch1/ch2/ch3`  | 多通道采样数据              |
| `crc`          | 校验字段，当前使用 XOR 校验 |

## 解析流程

1. 串口接收到原始字节流
2. 按换行拆出完整帧
3. 检查帧头是否为 `$DATA`
4. 提取 `*` 前的数据字段和 `*` 后的校验值
5. 计算 XOR 校验并与帧内校验值比较
6. 校验通过后解析序号、时间戳和通道值
7. 生成 `ProtocolFrame`
8. 将通道值送入波形显示，并将完整帧送入日志和 CSV 记录模块

## 当前实现能力

- 协议帧按行解析
- XOR 校验
- 半包缓存
- 粘包逐帧处理
- 错误帧提示
- 基础丢包统计
- 协议帧日志输出
- 协议帧接入 CSV 保存

## 异常处理

| 异常情况     | 处理方式                        |
| ------------ | ------------------------------- |
| 帧头错误     | 按普通 ASCII 数据处理或直接忽略 |
| 校验失败     | 不进入波形和 CSV，输出错误日志  |
| 字段数量不足 | 丢弃该帧                        |
| 数值转换失败 | 丢弃该帧，避免通道错位          |
| 半包         | 暂存在缓冲区，等待下一批数据    |
| 粘包         | 按换行拆分后逐帧解析            |
| 序号跳变     | 记录丢包统计                    |

## 上位机命令帧

后续可以扩展上位机向 STM32 下发控制命令：

```text
$CMD,<target>,<value>*<crc>\r\n
```

示例：

```text
$CMD,LED,1*2B
$CMD,PWM,500*4F
$CMD,PERIOD,20*61
```
