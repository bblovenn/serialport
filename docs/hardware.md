# STM32 联调说明

## 联调目标

STM32 通过 UART 周期性上传多通道采样数据，Qt 上位机负责接收、解析、绘图、日志显示和 CSV 保存。

这个文档记录的是一套最小可复现的联调方案，方便你快速把项目跑起来，也方便后续复用到自己的 MCU 项目里。

## 硬件连接

| STM32 | USB-TTL | 说明 |
| --- | --- | --- |
| TX | RX | STM32 发送数据到 PC |
| RX | TX | PC 向 STM32 下发控制命令 |
| GND | GND | 共地 |
| 3.3V | 3.3V / 不接 | 视供电方案决定 |

注意：

- UART 通信需要 `TX/RX` 交叉连接
- 必须保证 STM32 与 USB-TTL 共地
- 如果 USB-TTL 不是 3.3V 电平，请先确认兼容性

## 串口参数

- 波特率：`115200`
- 数据位：`8`
- 校验位：`None`
- 停止位：`1`
- 流控：`None`

## 数据上传示例

STM32 每 20 ms 发送一帧：

```text
$DATA,15,123456,25.60,60.20,3.31*7A
```

示例通道定义：

| 通道 | 含义 |
| --- | --- |
| `ch1` | ADC 电压 |
| `ch2` | 温度 |
| `ch3` | 电位器百分比 |

## 上位机命令示例

```text
$CMD,LED,1*2B
$CMD,PWM,500*4F
$CMD,PERIOD,20*61
```

| 命令 | 说明 |
| --- | --- |
| `LED` | 控制 LED 开关 |
| `PWM` | 设置 PWM 参数 |
| `PERIOD` | 设置上传周期，单位 ms |

## 已验证环境

当前已经完成一轮真实联调，环境如下：

- STM32 开发板：使用 `USART1`
- 串口引脚：`PA9(TX)` / `PA10(RX)`
- USB 转串口模块：`CH340`
- 下载器：`ST-Link V2`
- 上位机系统：Windows
- 上位机串口：`COM11`
- 波特率：`115200`

## 当前接线

本次验证使用的最小串口接线如下：

| STM32 USART1 | CH340 |
| --- | --- |
| `PA9 (TX)` | `RXD` |
| `PA10 (RX)` | `TXD` |
| `GND` | `GND` |

说明：

- `TX/RX` 使用交叉连接
- `GND` 必须共地
- `CH340` 在这里作为串口桥接使用，不依赖它给 STM32 供电

## 已验证结果

本次联调已确认：

- STM32 可以周期发送 `$DATA,<seq>,<timestamp_ms>,<ch1>,<ch2>,<ch3>*<crc>` 协议帧
- Qt 上位机可以正确接收并完成 XOR 校验
- 协议帧可以正常进入波形绘制链路
- “开始记录 / 停止记录” 可以正常控制 CSV 保存
- CSV 文件中可保存：
  - `host_time`
  - `sequence`
  - `device_time_ms`
  - `ch1/ch2/ch3`
  - `extra_values`
  - `raw_frame`

## 演示流程

1. 连接 STM32、CH340 和 PC
2. 使用 `ST-Link V2` 下载程序到 STM32
3. 确认 Windows 识别出 `CH340` 对应串口，例如 `COM11`
4. 打开 Qt 上位机，选择对应串口和 `115200` 波特率
5. 打开串口，观察协议日志和实时波形
6. 点击“开始记录”，验证 CSV 保存
7. 点击“停止记录”或关闭串口，确认记录流程正常收尾
